## Scope

Analyze an approved `SourceSlice` and produce two versioned artifacts:

1. **`SourceSchemaArtifact`** — per-column shape: name, inferred SQL type, nullability,
   max observed length. One artifact per `(source_definition_id, source_slice_version)`.

2. **`SourceValueSummary`** — distinct values + counts per field. Capped at 500 distinct
   values per field. One summary per `(source_definition_id, source_slice_version, field_name)`.

Both are generated by sending a sample of rows (up to 200) to the AI adapter with a
schema-extraction prompt. The AI returns structured JSON validated against Pydantic models.

## New models (migration 0011)

```python
class SourceSchemaArtifact(Base):
    __tablename__ = "source_schema_artifacts"
    schema_artifact_id: str  # UUID pk
    source_definition_id: str  # FK → source_definitions
    source_slice_version: str
    columns: list[dict]  # [{name, inferred_type, nullable, max_length}]
    created_at: datetime

class SourceValueSummary(Base):
    __tablename__ = "source_value_summaries"
    summary_id: str  # UUID pk
    source_definition_id: str  # FK → source_definitions
    source_slice_version: str
    field_name: str
    value_counts: dict  # {value: count}, capped at 500 distinct values
    created_at: datetime
```

## API routes (`routes/analysis.py`)

```
POST /projects/{project_id}/sources/{source_definition_id}/analyze
  — trigger analysis on the latest approved SourceSlice
  — returns 202 with {"schema_artifact_id": "...", "status": "queued"}
  — central_team only

GET  /projects/{project_id}/sources/{source_definition_id}/schema
  — latest SourceSchemaArtifact for this source
  — returns columns list

GET  /projects/{project_id}/sources/{source_definition_id}/value-summary
  — SourceValueSummary records for the latest analyzed slice
  — supports ?field=<name> filter
```

## AI prompt structure

System prompt for schema extraction:
```
You are a data analyst. Given CSV or fixed-length record samples,
infer column schemas. Return a JSON object matching the schema below.
```

User message: first 200 rows as CSV text (or fixed-length lines with column positions
from `layout_information`).

Response model:
```python
class ColumnSchema(BaseModel):
    name: str
    inferred_type: Literal["text", "integer", "decimal", "date", "boolean", "uuid"]
    nullable: bool
    max_length: int | None

class AnalysisResult(BaseModel):
    columns: list[ColumnSchema]
```

## Notes

- Analysis is synchronous in this slice (same as runs execution)
- `layout_information` on `SourceDefinition` provides column positions for fixed-length files
- Results feed 001w (mapping) and 001x (lookup value mapping) — `SourceSchemaArtifact`
  columns drive the field-binding proposal; `SourceValueSummary` provides source values
  for the lookup entry UI
- PII masking rule: field names matching `PII_FIELD_NAMES` are masked with `***` in
  `SourceValueSummary.value_counts` before storage
